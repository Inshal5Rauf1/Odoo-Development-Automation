    Agent role and workflow:

    - **Phase 1: Clone & Analyze** — Accept module name + repo from $ARGUMENTS or from odoo-search agent handoff. Run `odoo-gen-utils extend-module {module} --repo {repo} --output-dir {output_dir}` to clone + analyze. Present the ModuleAnalysis summary to the user.

    - **Phase 2: Gap Analysis** — If a spec.json exists (from /odoo-gen:plan or from odoo-search agent's REFN-03 output):
      1. Read the refined spec.json
      2. Compare spec models/fields against ModuleAnalysis model_names/model_fields
      3. Produce structured gap report:
         - **Covered:** models/fields already in original module
         - **Missing:** spec requirements to implement in companion module
         - **Conflicts:** architectural mismatches needing resolution
      4. If coverage < 40%: recommend building from scratch per CLAUDE.md lesson

    - **Phase 3: Delta Code Generation (FORK-03)** — Generate the companion `_ext` module:
      1. **__manifest__.py:** depends includes original module name. Standard OCA format per knowledge base. Version 17.0.1.0.0. Data files list only new files.
      2. **models/*.py:** For extending existing models: use `_inherit = '{original.model}'` to add new fields. For new models: use `_name = '{new.model}'` with full field definitions. One Python file per model (OCA convention).
      3. **views/*.xml:** For extending existing views: use `<record>` with `inherit_id` ref and `<xpath>` expressions. Reference original module's view XML IDs. For new model views: standard form/tree/search views.
      4. **security/ir.model.access.csv:** Only for new models (not inherited fields on existing models).
      5. **security/security.xml:** Only if new security groups needed.
      6. **tests/*.py:** Test new fields and methods only. Use TransactionCase. Import original module's test utilities if available.
      7. **__init__.py files:** Correct import chains for models/ and any other subdirectories.

    - **Phase 4: Save Refined Spec (REFN-03)** — After generation:
      1. Save refined spec to `{module_name}_ext/spec.json` AND overwrite the original spec.json path (REFN-03: refined spec is the new source of truth for all downstream generation commands including render-module, validate, etc.)
      2. The refined spec excludes already-covered fields and focuses on the delta

    - **Phase 5: Verify** — After generation:
      1. List all created files in companion module
      2. Verify __manifest__.py has correct depends
      3. Verify _inherit references match original module's model names
      4. Verify xpath expressions reference correct view XML IDs
      5. Optionally run `odoo-gen-utils validate {module}_ext/` if validation infra available

    Key Odoo patterns the agent MUST follow:
    - NEVER modify files in the original cloned module directory
    - Use `_inherit` for adding fields to existing models (not `_name` + `_inherit` which creates new model)
    - Use `xpath` with `position="after"|"before"|"inside"|"replace"` for view extensions
    - Use `inherit_id` with `ref="{original_module}.{view_xml_id}"` for view inheritance
    - Companion module category should match original module's category
    - Use explicit `groups=` on new fields if original module has security groups
