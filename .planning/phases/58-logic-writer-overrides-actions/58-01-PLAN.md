    1. Add 4 new fields to StubContext frozen dataclass (after error_messages):
       ```python
       stub_zone: dict[str, Any] | None = None
       exclusion_zones: tuple[dict[str, Any], ...] = ()
       action_context: dict[str, Any] | None = None
       cron_context: dict[str, Any] | None = None
       ```

    2. Add `_build_action_context(stub: StubInfo, model: dict, spec: dict) -> dict[str, Any] | None`:
       - Only called when method_type == "action"
       - Parse spec model's `workflow_states` or `states` for state definitions -> list of state names
       - Parse spec model's `business_rules` for transition rules -> list of transition dicts
       - Parse spec model's `approval_levels` for role-based transitions
       - Build `full_state_machine` with `states` and `transitions` lists
       - Extract `side_effects` from business rules mentioning "notification", "email", "create record", "log"
       - Extract `preconditions` from business rules mentioning "cannot", "must have", "required"
       - Return None if no state machine data found (graceful degradation)

    3. Add `_build_cron_context(stub: StubInfo, model: dict, spec: dict) -> dict[str, Any] | None`:
       - Only called when method_type == "cron"
       - Look up spec `cron` or `scheduled_actions` section matching this method
       - Extract domain_hint from business rules (convert natural language conditions to Odoo domain hint)
       - Classify processing_pattern using keyword matching:
         - "for each", "per record" -> "batch_per_record"
         - "generate", "create", "schedule" -> "generate_records"
         - "archive", "expire", "older than", "delete" -> "cleanup"
         - "calculate", "summarize", "report", "aggregate" -> "aggregate"
         - Default: "batch_per_record"
       - batch_size_hint: 100 (default), override if spec mentions batch size
       - error_handling: "log_and_continue" (default)
       - Return None if no cron data found

    4. Add `_build_stub_zones_for_override(stub: StubInfo, module_dir: Path) -> tuple[dict | None, tuple]`:
       - Read the source file (module_dir / stub.file)
       - Call `_find_stub_zones()` from stub_detector
       - For create(): find zones and assign position "pre_super" to first, "post_super" to second
       - For write(): find zone and assign position "post_super"
       - Build exclusion_zones from surrounding template code (lines outside markers within the method)
       - Return (stub_zone_for_this_stub, exclusion_zones_tuple)
       - NOTE: stub_zone in the context represents the zone relevant to THIS stub. If create() has 2 zones, the report will have 2 separate entries (one per zone).

    5. Update `build_stub_context()`:
       - After existing method_type classification, add:
       - If method_type == "action": call _build_action_context
       - If method_type == "cron": call _build_cron_context
       - If method_type == "override": call _build_stub_zones_for_override (only when module_dir is available -- add optional module_dir parameter)
       - Pass new fields to StubContext constructor

    **Report serialization (report.py):**

       if context.stub_zone:
