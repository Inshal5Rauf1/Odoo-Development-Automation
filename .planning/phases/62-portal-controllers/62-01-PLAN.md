  <behavior>
    - E23 detects ownership path "student_id" that terminates at uni.student (not res.users) -> error
    - E23 accepts direct path "user_id" that terminates at res.users -> no error
    - E23 accepts multi-hop path "student_id.user_id" traversing uni.student -> res.users -> no error
    - E23 reports error when intermediate field does not exist in model
    - E23 uses model registry for cross-module resolution when available
    - E23 falls back to spec models for same-module resolution
    - E23 is called from semantic_validate() and errors appear in result
    - E23 skips validation gracefully when models cannot be resolved (LOW confidence, warning not error)
  </behavior>
  <action>
    1. Add `_check_e23()` function to `semantic.py`:
       - Signature: `_check_e23(module_dir: Path, spec: dict[str, Any] | None = None, registry: "ModelRegistry | None" = None) -> list[ValidationIssue]`
       - For each page in spec.get("portal", {}).get("pages", []):
         - Parse ownership path by splitting on "."
         - Starting from page["model"], traverse each hop:
           - Look up the model in spec models first (by name), then in registry
           - Find the field at current hop in the model's fields
           - If field not found, emit E23 error: "Ownership path '{ownership}' on page '{page_id}': field '{hop}' not found on model '{current_model}'"
           - If field is relational (Many2one/One2many/Many2many), get comodel_name and continue to next hop
         - After all hops traversed, check the LAST field's comodel_name == "res.users" (for relational) or the last field is on res.users (for terminal)
         - If path does not terminate at res.users, emit E23 error: "Ownership path '{ownership}' on page '{page_id}' does not terminate at res.users (terminates at '{terminal_model}')"
         - If model cannot be resolved at any hop, emit W7 warning: "Cannot validate ownership path '{ownership}' on page '{page_id}' -- model '{model}' not in spec or registry"
       - Helper `_resolve_model_fields(model_name, spec, registry)` returns dict of field_name -> field_info or None

    2. Wire `_check_e23` into `semantic_validate()`:
       - After E17 check block, add:
         ```python
         # E23: Portal ownership path validation
         if spec is not None:
             result.errors.extend(_check_e23(output_dir, spec, registry))
         ```
       - Note: semantic_validate currently does not receive `spec`. Add an optional `spec: dict[str, Any] | None = None` parameter to `semantic_validate()` signature (default None, backward compatible).
