<task type="auto">
  <name>Task 2: Create management script and XML-RPC smoke test</name>
  <files>scripts/odoo-dev.sh, scripts/verify-odoo-dev.py</files>
  <action>
Create scripts/ directory and two files:

1. **scripts/odoo-dev.sh** -- Lifecycle management script. Must be executable (chmod +x).

   Use `set -euo pipefail`. Resolve SCRIPT_DIR and PROJECT_ROOT. Set COMPOSE_FILE to `$PROJECT_ROOT/docker/dev/docker-compose.yml`. Read DB_NAME from `${ODOO_DEV_DB:-odoo_dev}`. Set MODULES to `base,mail,sale,purchase,hr,account`.

   Helper functions:
   - `_compose()`: runs `docker compose -f "$COMPOSE_FILE" "$@"`
   - `_db_exists()`: checks if database exists by running `psql -U odoo -tAc "SELECT 1 FROM pg_database WHERE datname='$DB_NAME'"` via `_compose exec -T db` and grepping for 1. Returns exit code.
   - `_init_modules()`: runs `_compose run --rm -T odoo odoo -d "$DB_NAME" -i "$MODULES" --stop-after-init --no-http --log-level=warn`. MUST use `run --rm` NOT `exec` (exec causes serialization failures -- see CLAUDE.md mistake #4). Prints progress messages before and after.
   - `_wait_healthy()`: loops up to 60 seconds checking `_compose ps --format json` for odoo service health status. Prints "Waiting for Odoo to be healthy..." and dots. Exits with error if timeout.

   Commands (via case statement on `${1:-help}`):
   - `start`: Print starting message. Run `_compose up -d --wait db`. Check `_db_exists`; if false, call `_init_modules`. Then `_compose up -d --wait`. Print ready message with URL `http://localhost:${ODOO_DEV_PORT:-8069}`.
   - `stop`: Run `_compose down`. Print stopped message.
   - `status`: Run `_compose ps`. Then attempt to run `python3 "$SCRIPT_DIR/verify-odoo-dev.py"` if it exists, suppressing errors.
   - `reset`: Print warning. Run `_compose down -v`. Print message that data is destroyed and user should run start to reinitialize.
   - `logs`: Run `_compose logs -f "${2:-odoo}"`.
   - `*` (help): Print `Usage: $0 {start|stop|status|reset|logs}`.

2. **scripts/verify-odoo-dev.py** -- XML-RPC connectivity smoke test. Standalone Python script (no project dependencies, stdlib only).

   Function `verify_xmlrpc(url, db, username, password)` with defaults from env vars: url from `ODOO_URL` or `http://localhost:{ODOO_DEV_PORT or 8069}`, db from `ODOO_DEV_DB` or `odoo_dev`, username from `ODOO_DEV_USER` or `admin`, password from `ODOO_DEV_PASSWORD` or `admin`.

   Steps:
   a. Create `xmlrpc.client.ServerProxy(f"{url}/xmlrpc/2/common")` and call `version()`. Print server version.
   b. Call `authenticate(db, username, password, {})`. Assert uid is truthy. Print uid.
   c. Create `xmlrpc.client.ServerProxy(f"{url}/xmlrpc/2/object")`. Call `execute_kw` to `search_count` on `ir.model` (empty domain). Print model count.
   d. Call `execute_kw` to `search_read` on `ir.module.module` with domain `[["state", "=", "installed"]]` and fields `["name"]`. Check that {base, mail, sale, purchase, hr, account} are all present. Print installed count and any missing modules.
   e. Return True if all checks pass, False otherwise.

   Error handling: catch `ConnectionRefusedError` (print "Connection refused, is Odoo running?"), `xmlrpc.client.Fault` (print fault string), and generic `Exception` (print error).

   `if __name__ == "__main__"`: call `verify_xmlrpc()`, `sys.exit(0 if success else 1)`.

   Use os.environ.get() for all env var reads. Add a shebang `#!/usr/bin/env python3`.

After creating both files, run `chmod +x scripts/odoo-dev.sh scripts/verify-odoo-dev.py`.
  </action>
