    - Test: field with explicit full external ID `groups: "other_module.some_group"` is kept as-is (no resolution)
    - Test: non-sensitive field without groups key is left unchanged
    - Test: view_fields referencing a restricted field get `view_groups` key matching the field's `groups` value
    - Test: view_fields referencing a non-restricted field do NOT get `view_groups`

    - Test: legacy fallback (no security block) -- sensitive fields get `"{module}.group_{module}_manager"` (legacy manager group)
  </behavior>
  <action>
    **In preprocessors.py**, add two new helpers to `_process_security_patterns`:

    1. `_security_enrich_fields(spec, security_roles)` -- Walk all models' fields:
       - If `field.get("sensitive")` and no `groups` key: set `field["groups"]` to `"{module_name}.{highest_role.xml_id}"`
       - If `field.get("groups")` is a bare role name (exists in roles list): resolve to `"{module_name}.group_{module_name}_{role_name}"`
       - If `field.get("groups")` contains a dot (already a full external ID): leave as-is
       - Return new spec with enriched fields (immutable -- do NOT mutate input)

    2. `_security_auto_fix_views(spec)` -- Walk all models' `view_fields` (or `fields` list since view_fields is computed later in context builder):
       - Build a set of restricted field names (fields that have `groups` key)
       - For each field in each model's fields list: if `field["name"]` is in restricted set, add `view_groups` key with the same groups value
       - Log at INFO level: `"Auto-applied groups='%s' to field '%s' in views for model '%s'"` for each auto-fixed field
       - Return new spec with enriched fields (immutable)

    Update `_process_security_patterns` main function to call these two helpers after `_security_build_acl_matrix` and `_security_detect_record_rule_scopes`:
    ```python
    spec = _security_enrich_fields(spec, roles)
    spec = _security_auto_fix_views(spec)
    ```

    The same logic also applies in `_inject_legacy_security` -- call `_security_enrich_fields` and `_security_auto_fix_views` there too, using the legacy roles.

    **In test_renderer.py**, add `TestSecurityFieldGroups` and `TestSecurityViewAutoFix` test classes with the behaviors listed above. Use `_make_security_spec()` helper from Plan 01 and extend it with sensitive fields.
