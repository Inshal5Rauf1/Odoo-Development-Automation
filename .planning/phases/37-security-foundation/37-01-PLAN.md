    **In preprocessors.py**, add `_process_security_patterns(spec)` and these helpers (all prefixed `_security_`):

    1. `_security_validate_spec(security)` -- Validates that `defaults` keys exactly match `roles` array. Validates CRUD strings contain only `c`, `r`, `u`, `d` (lowercase). Raises `ValueError` on mismatch.

    2. `_parse_crud(crud_str: str) -> dict[str, int]` -- Converts "cru" to `{"perm_create": 1, "perm_read": 1, "perm_write": 1, "perm_unlink": 0}`. Normalizes to lowercase first. Raises ValueError if invalid chars.

    3. `_security_build_roles(module_name, security)` -- Builds list of role dicts:
       ```python
       [{"name": "viewer", "label": "Viewer", "xml_id": "group_{module}_{role}",
         "implied_ids": "base.group_user" (for lowest) or "group_{module}_{prev_role}" (for others),
         "is_highest": bool}]
       ```
       Roles ordered lowest-to-highest per `security["roles"]` array order.

    4. `_security_build_acl_matrix(spec, security)` -- For each model, build `security_acl` list. Check `security.acl[model_name]` first; if not present, use `security.defaults`. Parse CRUD string via `_parse_crud`. Returns enriched models.

    5. `_security_detect_record_rule_scopes(model)` -- Auto-detect scopes:
       - "ownership" if any field named `user_id` with type `Many2one` or `create_uid` exists
       - "department" if any field named `department_id` exists
       - "company" if any field named `company_id` with type `Many2one` exists
       Returns list of scope strings. If model has `record_rules` key in spec, use that instead (override).

    6. `_inject_legacy_security(spec)` -- When no `security` block exists, inject legacy 2-role system:
       - `security_roles`: user (implied_ids=base.group_user) and manager (implied_ids=group_{module}_user, is_highest=True)
       - Each model gets `security_acl` with user=cru and manager=crud defaults
       - Record rule scopes auto-detected same as spec-driven path

    Main entry point `_process_security_patterns(spec)`:
    - If no `security` block: call `_inject_legacy_security` and return
    - Validate spec, build roles, build ACL matrix, detect record rule scopes
    - Return `{**spec, "security_roles": roles}` with enriched models

    {# security_group.xml.j2 -- Odoo security groups and module category #}
    <?xml version="1.0" encoding="utf-8"?>
    <odoo>

        <!-- Module Category -->
        <record id="module_category_{{ module_name }}" model="ir.module.category">
            <field name="name">{{ module_title }}</field>
            <field name="sequence">100</field>
        </record>

        <!-- {{ role.label }} Group{% if role.is_highest %} (highest){% endif %} -->

            <field name="category_id" ref="module_category_{{ module_name }}"/>
            <field name="implied_ids" eval="[(4, ref('{{ role.implied_ids }}'))]"/>
    {% if role.is_highest %}
            <field name="users" eval="[(4, ref('base.user_root')), (4, ref('base.user_admin'))]"/>
    {% endif %}
        </record>

    {% endfor %}
    </odoo>
