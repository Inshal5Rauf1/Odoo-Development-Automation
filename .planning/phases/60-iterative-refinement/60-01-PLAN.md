  <action>
    **conflict.py:**
    - Frozen dataclass `ConflictResult` with `safe_to_overwrite: tuple[str, ...]`, `conflicts: tuple[str, ...]`, `stub_mergeable: tuple[str, ...]` (tuples for immutability).
    - `detect_conflicts(manifest: GenerationManifest, affected_files: list[str], module_dir: Path, skeleton_dir: Path | None = None) -> ConflictResult`
    - For each affected file:
      1. Find matching ArtifactEntry in `manifest.artifacts.files` by path
      2. If not in manifest -> treat as safe (new file, no prior version to conflict with)
      3. If file doesn't exist on disk -> skip (will be created fresh)
      4. Compute current SHA256 via `compute_file_sha256(module_dir / path)`
      5. If `manifest_hash == current_hash` -> safe_to_overwrite
      6. If hashes differ -> check stub zones:
         - Read current file content and skeleton content (if skeleton_dir exists)
         - If skeleton_dir and skeleton file exists: compare skeleton lines vs current lines
         - If ALL differences are within BUSINESS LOGIC START/END zones -> stub_mergeable
         - Otherwise -> conflicts
    - For stub zone detection, use `_find_stub_zones()` from `logic_writer.stub_detector`.
    - Compare skeleton vs current line-by-line. If every differing line falls within a stub zone range -> stub_mergeable.

    **merge.py:**
    - `extract_filled_stubs(source_lines: list[str]) -> list[dict]` — finds stub zones using `_find_stub_zones()`, then for each zone extracts the method name by scanning upward from `start_line` to find the `def ` line. Returns list of `{"method_name": str, "content_lines": list[str], "start_line": int, "end_line": int}`. Content lines are the lines BETWEEN markers (exclusive of markers themselves). If zone content is just `pass` or a TODO comment, treat as unfilled and skip.
    - `inject_stubs_into(new_structure: str, filled_stubs: list[dict]) -> str` — parses new_structure into lines, finds stub zones using `_find_stub_zones()`, for each zone extracts method name, looks up matching filled_stub by method_name, replaces zone content with filled implementation. Returns merged string. Unmatched zones left as-is (new methods not previously implemented).
    - Match by method name: scan backward from start_line in new_structure to find `def method_name(` — extract method_name from that line.
