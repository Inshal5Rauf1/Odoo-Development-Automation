  <action>
    **renderer.py changes:**

    Add new keyword parameters to `render_module()`:
    ```python
    def render_module(
        spec: dict[str, Any],
        template_dir: Path,
        output_dir: Path,
        verifier=None,
        *,
        no_context7: bool = False,
        fresh_context7: bool = False,
        hooks: list | None = None,
        resume_from: "GenerationManifest | None" = None,
        force: bool = False,
        dry_run: bool = False,
    ) -> "tuple[list[Path], list[VerificationWarning]]":
    ```

    Add iterative mode detection AFTER Pydantic validation + cycle check, BEFORE preprocessing:
    1. Save the raw spec (before preprocessing) for stash comparison.
    2. Load spec stash: `old_spec = load_spec_stash(module_dir)`.
    3. If `old_spec is not None and not force`:
       a. `diff_result = compute_spec_diff(old_spec, spec_raw)` (compare raw specs)
       b. If `diff_result is None`: log "Spec unchanged. Nothing to do. Use --force to regenerate." and return `([], [])`.
       c. `affected = determine_affected_stages(diff_result)`
       d. If `dry_run`: format and log the diff summary + affected stages + affected files, return `([], [])`.
       e. Load existing manifest: `existing_manifest = load_manifest(module_dir)`.
       f. After preprocessing runs, detect affected files from the affected stages.
    4. If iterative mode active, filter the `stages` list: `stages = [(name, fn) for name, fn in stages if name in affected.stages]`.
    5. Before overwriting files, run conflict detection: `conflicts = detect_conflicts(existing_manifest, affected_file_paths, module_dir, skeleton_dir)`.
    6. For `conflicts.stub_mergeable` files: after re-rendering, call `extract_filled_stubs()` on the CURRENT file, then `inject_stubs_into()` with the newly rendered content, write the merged result.
    7. For `conflicts.conflicts` files: write the new version to `.odoo-gen-pending/{relative_path}` instead of overwriting. Log a message.
    8. After all stages: `save_spec_stash(spec_raw, module_dir)` to update the stash.
    9. For iterative mode manifest handling: start from the existing manifest and update only the stages that re-ran. Use `session.to_manifest()` but merge with existing manifest's stages for skipped stages.

    **Important implementation details:**
    - `spec_raw` is captured BEFORE `validate_spec()` and `run_preprocessors()` calls (the raw input dict).
    - The `module_dir` is computed from `output_dir / spec["module_name"]` which happens early enough.
    - `skeleton_dir` is `output_dir / ".odoo-gen-skeleton" / module_name`.
    - The conflict detection + merge happens WITHIN the stage loop, wrapping each stage_fn call when iterative mode is active. The cleanest approach: run stage_fn normally, then for each file it produced, check conflict status and handle accordingly. This avoids modifying individual stage functions.
    - For MODEL_REMOVED: move affected files to `.odoo-gen-pending/removed/` with a log message. Do NOT physically delete.
    - The `.odoo-gen-pending/` directory is created under the module_dir.

    **cli.py changes:**
    - Add `--force` flag: `@click.option("--force", is_flag=True, default=False, help="Full regeneration, ignore existing manifest")`
    - Add `--dry-run` flag: `@click.option("--dry-run", "dry_run", is_flag=True, default=False, help="Show what would change without writing files")`
    - Pass both to `render_module()`.
    - For dry-run output: print the diff categories, affected stages, and files that would be updated.
    - When conflicts exist: print summary of pending files with instructions to use `odoo-gen resolve`.

  <action>
    **resolve.py:**
    - `PENDING_DIR_NAME = ".odoo-gen-pending"` constant.
    - `resolve_status(module_dir: Path) -> list[str]` — lists all files in `module_dir / PENDING_DIR_NAME` recursively, returns relative paths. Returns empty list if dir doesn't exist.
    - `resolve_accept_new(module_dir: Path, relative_path: str) -> bool` — copies `module_dir / PENDING_DIR_NAME / relative_path` to `module_dir / relative_path`, removes pending file. Loads manifest, updates the artifact entry's SHA256 for the resolved file, saves manifest. Returns True if resolved, False if pending file not found.
    - `resolve_keep_mine(module_dir: Path, relative_path: str) -> bool` — removes `module_dir / PENDING_DIR_NAME / relative_path` without overwriting. Returns True if removed, False if not found.
    - `resolve_accept_all(module_dir: Path) -> int` — calls `resolve_accept_new()` for each pending file. Returns count of resolved files.
    - `_cleanup_pending(module_dir: Path)` — if `.odoo-gen-pending/` is empty after resolution, remove the directory. Called after each resolve operation.
    - All functions handle the `removed/` subdirectory: `resolve_accept_new` on a removed file means confirming deletion (remove from module_dir too). `resolve_keep_mine` on a removed file means keeping the file in module_dir.
