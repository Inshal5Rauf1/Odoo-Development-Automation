    1. Create `src/odoo_gen/scaffold/spec.py`:
       - `ModuleSpec(BaseModel)` (NOT frozen -- it is a data container for scaffold input, not pipeline state).
       - Fields: `module_name: str` (technical name, snake_case), `module_title: str` (human-readable), `module_description: str = "A sample Odoo module"`, `odoo_version: str = "17.0"`, `version: str = ""` (computed from odoo_version if empty: e.g., "17.0.1.0.0"), `author: str = "Your Company"`, `license: str = "LGPL-3"`, `category: str = "Uncategorized"`, `model_name: str = ""` (computed from module_name: e.g., "my_module.sample"), `model_class: str = ""` (computed: e.g., "MyModuleSample"), `model_description: str = "Sample Record"`.
       - Add a `@model_validator(mode="after")` that computes `version`, `model_name`, and `model_class` from `module_name` and `odoo_version` if they are empty.
       - Add a `@field_validator("module_name")` that validates: lowercase, underscores only (no dashes/spaces), starts with a letter, not a Python keyword. Auto-convert if possible ("My Cool Module" -> "my_cool_module").
       - Add a classmethod `from_interactive(name: str, odoo_version: str = "17.0", **kwargs) -> ModuleSpec` that constructs a spec with sensible defaults derived from the name.

    2. Create `src/odoo_gen/scaffold/renderer.py`:
       - `create_jinja_env() -> Environment` that creates a Jinja2 Environment with `FileSystemLoader` pointing to the templates directory. Use `importlib.resources.files("odoo_gen") / "templates"` to resolve the template path at runtime (NOT `__file__` based). Set `keep_trailing_newline=True`, `trim_blocks=True`, `lstrip_blocks=True`.
       - `render_template(env: Environment, template_name: str, context: dict) -> str` that renders a template with the given context and returns the string.

    3. Create 8 Jinja2 template files in `src/odoo_gen/templates/`:

       **`__manifest__.py.j2`:** OCA-compliant manifest with `name`, `summary`, `version` ({{ version }}), `category`, `author`, `website`, `license`, `depends` (["base"]), `data` (list of view/security XML/CSV files), `installable` (True). Use correct Odoo 17.0 format. Include comments explaining each field.

       **`__init__.py.j2`:** Root `__init__.py` that imports the `models` package. Content: `from . import models`.

       **`models/__init__.py.j2`:** Models package init that imports the sample model file. Content: `from . import {{ module_name.split('.')[-1] if '.' in module_name else 'sample' }}`.

       **`models/sample.py.j2`:** Odoo model class inheriting `models.Model`. Fields: `name` (Char, required), `description` (Text), `active` (Boolean, default=True), `value` (Float, digits=(16, 2)). Include `_name = "{{ model_name }}"`, `_description = "{{ model_description }}"`. Import from `odoo import api, fields, models`. NO `@api.multi`, NO `@api.one`, NO `openerp` imports.

       **`views/sample_views.xml.j2`:** Odoo 17.0 XML with form view, list view (using `<list>` NOT `<tree>`), search view, action, and menu item. Use inline `invisible`/`readonly` expressions, NOT `attrs={}`. Use `{% raw %}...{% endraw %}` around any Odoo domain expressions that conflict with Jinja2 syntax. Wrap all views in `<odoo><data>...</data></odoo>`. Each view record has an `id` derived from module_name.

       **`security/ir.model.access.csv.j2`:** CSV with header `id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink`. One row granting full access to `base.group_user` for the generated model. Model external ID format: `model_{{ model_name | replace('.', '_') }}`.

       **`tests/__init__.py.j2`:** Tests package init importing the test file. Content: `from . import test_{{ module_name.split('_')[-1] if '_' in module_name else 'sample' }}`.

       **`tests/test_sample.py.j2`:** A `TransactionCase` test class with one test method that creates a record and asserts it exists. Import from `odoo.tests.common import TransactionCase`. Test creates a record via `self.env["{{ model_name }}"].create({"name": "Test"})` and asserts the name is correct.

    4. Create `src/odoo_gen/scaffold/builder.py`:
       - `build_scaffold(spec: ModuleSpec, output_dir: Path) -> Path` function that:
         a. Creates the module directory: `output_dir / spec.module_name`
         b. Creates subdirectories: `models/`, `views/`, `security/`, `tests/`
         c. Creates a Jinja2 environment via `create_jinja_env()`
         d. Builds the template context dict from `spec.model_dump()`
         e. Renders and writes each template to its target path
         f. Returns the module directory path
       - The function should use the UI display functions: call `announce_stage("Scaffolding module...")` at the start, `show_module_summary(...)` at the end.
       - File write order: manifest, root init, models init, model file, views, security CSV, tests init, test file.
       - Raise clear errors if output_dir does not exist or module directory already exists (with an option to overwrite).

    5. Create `src/odoo_gen/scaffold/__init__.py` exporting `ModuleSpec`, `build_scaffold`, `render_template`, `create_jinja_env`.

    6. Create `tests/unit/test_renderer.py`:

       - Test rendering the view template produces valid XML (use `xml.etree.ElementTree.fromstring()`).

       - Test the created XML file is well-formed (`ElementTree.parse()`).
