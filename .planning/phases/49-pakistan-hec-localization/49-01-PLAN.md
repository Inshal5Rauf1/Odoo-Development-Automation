  <files>python/src/odoo_gen_utils/preprocessors/pakistan_hec.py, python/tests/test_pakistan_hec.py, python/pyproject.toml</files>
  <behavior>
    Registration:
    - Preprocessor registered at order=25 with name="pakistan_hec"
    - run_preprocessors() includes pakistan_hec in pipeline between computation_chains and constraints
    - Non-pk spec (localization != "pk") returns spec unchanged
    - pk spec with model having empty/missing pk_fields injects nothing into that model
    - pk spec always sets extra_data_files: ["data/pk_currency_data.xml"]

    CNIC injection (pk_fields contains "cnic"):
    - Adds field: name="cnic", type="Char", size=15, copy=False, tracking=True, string="CNIC", help text
    - Adds SQL constraint: name="{model_var}_cnic_unique", definition="UNIQUE(cnic)", message="CNIC must be unique."
    - Adds complex_constraint entry: type="pk_cnic", name="cnic", fields=["cnic"], check_body contains re.sub normalize + 13-digit validation + XXXXX-XXXXXXX-X formatting
    - Does NOT add field if "cnic" field already exists in model
    - Does NOT add SQL constraint if "{model_var}_cnic_unique" already exists

    Phone injection (pk_fields contains "phone"):
    - Adds field: name="phone_pk", type="Char", string="Phone (PK)", tracking=True
    - Adds complex_constraint entry: type="pk_phone", name="phone_pakistan", fields=["phone_pk"], check_body contains try/except ImportError with phonenumbers parse+validate+format_number AND regex fallback (mobile + landline patterns)

    NTN injection (pk_fields contains "ntn"):
    - Adds field: name="ntn", type="Char", size=9, string="NTN", copy=False, tracking=True
    - Adds SQL constraint: name="{model_var}_ntn_unique"

    STRN injection (pk_fields contains "strn"):
    - Adds field: name="strn", type="Char", size=15, string="STRN", copy=False, tracking=True
    - Adds SQL constraint: name="{model_var}_strn_unique"

    HEC Registration (pk_fields contains "hec_registration"):
    - Adds field: name="hec_registration_no", type="Char", copy=False, tracking=True, string="HEC Registration No."
    - Adds SQL constraint: name="{model_var}_hec_reg_unique"

    GPA injection (pk_fields contains "gpa"):
    - Adds field: name="gpa", type="Float", digits=(3, 2), default=0.0, string="GPA"
    - Adds complex_constraint: type="pk_gpa", check_body validates 0.00-4.00

    Credit Hours injection (pk_fields contains "credit_hours"):
    - Adds field: name="credit_hours", type="Integer", default=3, string="Credit Hours"
    - Adds complex_constraint: type="pk_credit_hours", check_body validates 0-6

    Degree Level injection (pk_fields contains "degree_level"):
    - Adds field: name="degree_level", type="Selection", selection with 7 values (matriculation through postdoc), string="Degree Level"

    Recognition Status injection (pk_fields contains "recognition_status"):
    - Adds field: name="recognition_status", type="Selection", selection with 4 values, default="recognized", string="Recognition Status"

    Immutability:
    - Input spec dict is never mutated
    - Returns a new spec dict with enriched models

    Integration with template rendering:
    - Injected complex_constraints use check_body pattern (non-temporal type) so model.py.j2 renders full method bodies
    - needs_api set to True on models with pk constraints (constrained_fields triggers it in renderer_context)
    - needs_translate set to True on models with pk constraints (complex_constraints triggers it in renderer_context)
