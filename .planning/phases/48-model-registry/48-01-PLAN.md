  <action>
    Write tests FIRST in test_registry.py (RED). Then implement:

    1. **known_odoo_models.json** (~200 entries): Ship in python/src/odoo_gen_utils/data/. Structure:
       ```json
       {
         "_meta": { "odoo_version": "17.0", "model_count": N },
         "models": {
           "res.partner": { "module": "base", "fields": {"name": {"type": "Char"}, ...}, "is_mixin": false },
           ...
         }
       }
       ```
       Coverage: base (res.partner, res.users, res.company, res.country, res.currency, res.groups, etc.), mail (mail.thread, mail.activity.mixin, mail.message), hr basics (hr.employee, hr.department, hr.job), account basics (account.move, account.move.line, account.journal, account.account, account.tax), purchase/sale/stock basics, website, portal.mixin, product.product, product.template, uom.uom, ir.* models. Include commonly-referenced fields only (those needed for depends inference). Each entry: model name, key fields dict, owning module, is_mixin bool.

    2. **registry.py**: Create ModelRegistry class with dataclass-based internal structures:
       - `RegistryMeta` dataclass (version, last_updated, odoo_version, modules_registered)
       - `ModelEntry` dataclass (module, fields, inherits, mixins, description)
       - `ValidationResult` dataclass (errors, warnings, info lists + has_errors property)
       - `ModelRegistry` class with:
         - `__init__(self, registry_path: Path)` — initialize empty state
         - `load(self)` — read JSON file, populate _meta/_models/_dependency_graph (no-op if file doesn't exist)
         - `save(self)` — write current state to JSON file, update _meta.last_updated and modules_registered
         - `load_known_models(self)` — load known_odoo_models.json into _known_models dict
         - `register_module(self, module_name: str, spec: dict)` — extract models from spec, OVERWRITE any existing entries for this module, update dependency_graph
         - `remove_module(self, module_name: str)` — remove all models belonging to module, remove from dependency_graph
         - `validate_comodels(self, spec: dict) -> ValidationResult` — check all comodel_name references against known_models + registry models. Unknown = WARNING. Self-inherit = ERROR. Duplicate _name = ERROR.
         - `infer_depends(self, spec: dict) -> list[str]` — find owning modules of comodel targets in known_models and registry, exclude already-explicit depends
         - `detect_cycles(self) -> list[str]` — use graphlib.TopologicalSorter on _dependency_graph, return error messages
         - `list_modules(self) -> dict[str, list[str]]` — return {module_name: [model_names]}
         - `show_model(self, model_name: str) -> ModelEntry | None` — lookup model

       Use `from __future__ import annotations`. Use `graphlib.TopologicalSorter` for cycle detection (same pattern as preprocessors/validation.py). Use `json` for persistence. Use `dataclasses.asdict` for serialization. Use `datetime.now(timezone.utc).isoformat()` for timestamps.

    3. Run tests (GREEN). All tests must pass.
