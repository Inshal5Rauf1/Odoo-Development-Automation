    **_process_notification_patterns(spec) in preprocessors.py:**
    1. Find models with has_approval and approval levels containing notify objects
    2. For each such model, build notification_templates list:
       - Each template: xml_id (from notify.template), name (human readable), model_xml_id, subject, email_to (resolved from recipients), body_intro (derived from state transition), body_fields (auto-selected from model fields)
    3. Enrich approval_action_methods: for level[i], find the matching action method (action_approve_{state}) and add a `notification` sub-dict with template_xml_id and the resolved template reference
       - Level[0]'s notify enriches approval_submit_action (submit transitions INTO first level's state)
       - Level[1+]'s notify enriches the corresponding action_approve_{state} method
    4. Enrich approval_reject_action with notification from on_reject_notify if present
    5. Set has_notifications=True, needs_logger=True on model
    6. Set has_notification_models=True at module level (on spec)
    7. Add "mail" to spec["depends"] if not present
    8. Return new spec (pure function, no mutation)

    **Recipient resolution helper _resolve_recipient(recipient, module_name, security_roles):**
    - "creator" -> {"email_to": "{{ object.create_uid.partner_id.email }}"}
    - "role:{name}" -> {"email_to": "{{ ','.join(env.ref('{module}.group_{module}_{name}').users.mapped('email')) }}"}
    - "field:{field}" -> {"email_to": "{{ object.{field}.email }}"}
    - "fixed:{email}" -> {"email_to": "{email}"}

    **Body field selection helper _select_body_fields(model, max_fields=4):**
    - Exclude types: Binary, One2many, Many2many
    - Exclude names: create_uid, write_uid, create_date, write_date, message_ids, activity_ids, parent_path, state
    - Exclude computed fields
    - Priority: name/display_name first (always if present), then required fields, then fields with string attribute
    - Return list of dicts with name and label (from field "string" or name.replace("_"," ").title())

    4. Add "webhooks" to override_sources["create"] if on_create
    5. Add "webhooks" to override_sources["write"] if on_write non-empty
    6. Return new spec (pure function, no mutation)
