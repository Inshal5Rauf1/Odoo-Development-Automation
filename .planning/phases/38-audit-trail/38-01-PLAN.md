    - Test: audit.trail.log model gets security_acl with read-only for auditor+highest role, no access for others

  <action>
Add `_process_audit_patterns(spec: dict[str, Any]) -> dict[str, Any]` to preprocessors.py. Follow the pure-function pattern of existing preprocessors. Implementation details:

1. Scan models for `audit: true`. If none found, return spec unchanged.
2. For each audited model:
   - Build exclude set: ALWAYS_EXCLUDE (`message_ids`, `activity_ids`, `write_date`, `write_uid`) UNION spec's `audit_exclude` list UNION fields with type in SKIP_TYPES (`One2many`, `Many2many`, `Binary`)
   - Compute `audit_fields` = fields not in exclude set
   - Set `has_audit=True`, `audit_fields`, `audit_exclude` (sorted), `has_write_override=True`
   - Add `"audit"` to `override_sources["write"]` using `setdefault(defaultdict(set))` pattern
3. Synthesize `audit.trail.log` model via `_build_audit_log_model(module_name, audited_models)`:
   - Fields per CONTEXT.md: res_model (Char, indexed, required, readonly), res_id (Many2oneReference, model_field="res_model", readonly), changes (Json, readonly), user_id (Many2one comodel_name="res.users", required, readonly, index), operation (Selection write/create/unlink, required, readonly)
   - Set `_synthesized=True`, `_is_audit_log=True`, `chatter=False`, `audit=False`
4. Inject auditor role: If `audit: true` exists on any model AND "auditor" not in security_roles, inject an auditor role that implies `base.group_user` directly (sibling of lowest role, not in the hierarchy chain). Check `spec.get("security_roles", [])` for existing auditor.
5. Set ACL on audit.trail.log: `security_acl` list with read-only (perm_read=1, others=0) for auditor and highest role, no access (all 0) for all other roles.
6. Set `has_audit_log=True` on returned spec.

Write tests FIRST in test_preprocessors.py (RED), then implement (GREEN). Use `class TestAuditPreprocessor` test class.
