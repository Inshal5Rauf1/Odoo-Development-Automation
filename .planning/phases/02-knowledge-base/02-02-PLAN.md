<task type="auto">
  <name>Task 1: Create testing.md, actions.md, and inheritance.md</name>
  <files>knowledge/testing.md, knowledge/actions.md, knowledge/inheritance.md</files>
  <action>
Create three category files:

**knowledge/testing.md** (~250-350 lines) -- Test writing patterns:
- Section: Test Base Classes -- TransactionCase (most common), SavepointCase (deprecated in 17, use TransactionCase), HttpCase for controller tests
- Section: setUpClass Pattern -- `@classmethod`, `super().setUpClass()`, create test records, assign to cls attributes
- Section: CRUD Tests -- test create with valid data, test read, test write, test unlink, assert field values
- Section: Computed Field Tests -- create record, set dependency fields, assert computed value
- Section: Constraint Tests -- `with self.assertRaises(ValidationError)`, pass invalid data, verify constraint fires
- Section: Access Rights Tests -- create user with specific group, use `with_user()`, verify allowed/denied operations
- Section: Workflow Tests -- create record, call action methods, verify state transitions
- Section: Test File Organization -- one test file per model, `tests/__init__.py` imports, naming convention `test_model_name.py`
- Section: Changed in 17.0 -- SavepointCase deprecation, any test framework changes
- Section: Common Mistakes -- testing as admin (masks permission issues), not using setUpClass (slow tests), testing implementation not behavior

**knowledge/actions.md** (~200-250 lines) -- Actions and menus:
- Section: Window Actions -- `ir.actions.act_window`, required fields (name, res_model, view_mode), view_mode values (tree,form vs list,form)
- Section: Server Actions -- `ir.actions.server`, code actions, model method binding
- Section: Menu Hierarchy -- root menu, sub-menus, `parent_id`, `action` reference, `sequence` for ordering
- Section: Action-Model Binding -- linking action to model, domain filtering, context defaults
- Section: URL Actions -- `ir.actions.act_url`
- Section: Changed in 17.0 -- view_mode uses "tree" not "list" in 17.0
- Section: Common Mistakes -- orphan menus (no action), wrong view_mode values, missing menu sequence

**knowledge/inheritance.md** (~250-350 lines) -- Model and view inheritance:
- Section: Model Extension (_inherit) -- add fields to existing models, override methods, call super()
- Section: Model Delegation (_inherits) -- composition pattern, parent_id field
- Section: New Model from Existing (_name + _inherit) -- create new model based on existing one
- Section: View Inheritance -- `inherit_id`, xpath expressions, position attribute (inside, after, before, replace, attributes)
- Section: xpath Patterns -- `//field[@name='xxx']`, `//group`, `//page`, `//header`, `//sheet`
- Section: Priority and Sequence -- view `priority` field, inheritance order
- Section: Changed in 17.0 -- any inheritance-related changes
- Section: Common Mistakes -- forgetting super() in method override, wrong xpath expression, modifying instead of extending

Every rule: Rule statement, WRONG code, CORRECT code, Why explanation. Do NOT exceed 500 lines per file.

<task type="auto">
  <name>Task 2: Create data.md, i18n.md, controllers.md, and wizards.md</name>
  <files>knowledge/data.md, knowledge/i18n.md, knowledge/controllers.md, knowledge/wizards.md</files>
  <action>
Create four category files:

**knowledge/data.md** (~150-200 lines) -- Data files and demo data:
- Section: Data Files (`data/`) -- sequences (ir.sequence), default configuration records, email templates, scheduled actions (ir.cron)
- Section: Demo Data (`demo/`) -- sample records for testing and demonstration, `noupdate="1"` attribute, referencing existing records safely
- Section: XML Data Format -- `<record>` tag, `<field>` with name/ref/eval attributes, `eval="[(4, ref('xxx'))]"` for M2M
- Section: CSV Data Format -- when to use CSV vs XML, ir.model.access.csv pattern
- Section: Load Order in Manifest -- security groups -> ACLs -> data files -> views (in `data` list)
- Section: Changed in 17.0 -- any data-related changes
- Section: Common Mistakes -- referencing records from uninstalled modules, forgetting noupdate on demo data, wrong eval syntax

**knowledge/i18n.md** (~100-150 lines) -- Internationalization:
- Section: Translation Markup -- `_("string")` in Python code, wrapping user-facing strings
- Section: Translatable Fields -- `string` parameter on fields is auto-translatable, `translate=True` for Char/Text
- Section: .pot File Generation -- `odoo-bin --modules=module_name --i18n-export=module_name.pot`
- Section: XML Translations -- strings in XML views are auto-translatable
- Section: Changed in 17.0 -- any i18n changes
- Section: pylint-odoo Rules -- W8160 with explanation
- Section: Common Mistakes -- missing _() on user-facing strings, translating technical strings, hardcoded strings in templates

**knowledge/controllers.md** (~150-200 lines) -- HTTP controllers:
- Section: Controller Class -- inherit from `http.Controller`, class decoration
- Section: Route Decorators -- `@http.route()`, path patterns, auth options (user, public, none), type (http, json), methods, website flag
- Section: Request/Response -- `request.env`, `request.render()`, `request.make_response()`, JSON responses
- Section: Website Controllers -- `website=True`, template rendering, SEO URLs
- Section: Changed in 17.0 -- any controller-related changes
- Section: Common Mistakes -- missing auth parameter, not handling CSRF, accessing env without sudo when needed

**knowledge/wizards.md** (~200-250 lines) -- TransientModel patterns:
- Section: Wizard Model -- `models.TransientModel`, `_name`, `_description`, transient lifecycle (auto-cleaned)
- Section: Wizard Fields -- regular fields for user input, Many2one to pass context, default values from context
- Section: Wizard Action Method -- `action_confirm()` or similar, return action dict or close wizard (`{'type': 'ir.actions.act_window_close'}`)
- Section: Wizard View -- form view with `<footer>` for buttons, `special="cancel"` for close button
- Section: Launching Wizards -- server action or button with `type="action"`, passing `active_ids` via context
- Section: Changed in 17.0 -- any wizard-related changes
- Section: Common Mistakes -- wizard fields referencing wrong model, not passing context, forgetting to clean up transient records

Every rule: Rule statement, WRONG code, CORRECT code, Why explanation. Do NOT exceed 500 lines per file.
