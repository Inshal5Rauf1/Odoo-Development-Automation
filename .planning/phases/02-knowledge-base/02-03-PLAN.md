  <action>
Create extensibility mechanism:

**knowledge/custom/README.md** -- Instructions for adding custom rules:
- Explain the custom/ directory purpose: extend the shipped knowledge base with team-specific rules
- Explain file naming: match shipped file names (e.g., `models.md` adds model rules, `security.md` adds security rules)
- Explain format requirements: must follow the Rule + WRONG + CORRECT + Why format
- Explain loading order: agents load MASTER.md + shipped category + matching custom/ file
- Explain that custom rules EXTEND defaults, never override shipped rules
- Provide a minimal example of a valid custom rule file
- Explain how to validate: `odoo-gen-utils validate-kb --custom`

**python/src/odoo_gen_utils/kb_validator.py** -- Format validator for custom rules:
- Function `validate_kb_file(path: Path) -> dict` that returns `{"valid": bool, "errors": list[str], "warnings": list[str]}`
- Check 1: File is valid markdown (not empty, starts with `#` heading)
- Check 2: Contains at least one rule section (line starting with `### `)
- Check 3: Contains at least one code block (triple backtick)
- Check 4: Does not exceed 500 lines
- Check 5: Has no obvious structural issues (unclosed code blocks)
- Function `validate_kb_directory(path: Path) -> dict` that validates all .md files in a directory
- Format-only checks: no semantic validation of rule content

**python/src/odoo_gen_utils/cli.py** -- Add `validate-kb` subcommand:
- New Click command `validate_kb` registered with the CLI group
- `--custom` flag to validate only the custom/ directory
- `--all` flag to validate all knowledge base files
- Default (no flag) validates custom/ directory
- Prints validation results: file name, valid/invalid, error messages
- Exit code 0 if all valid, 1 if any invalid
- Knowledge base path: resolve from `~/.claude/odoo-gen/knowledge/` (installed location) or fall back to `./knowledge/` (dev location)
  </action>
