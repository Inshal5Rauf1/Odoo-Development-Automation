<task type="auto">
  <name>Task 1: Create MASTER.md and models.md</name>
  <files>knowledge/MASTER.md, knowledge/models.md</files>
  <action>
Create the `knowledge/` directory and two files:

**knowledge/MASTER.md** (~150-200 lines) -- Global Odoo 17.0 conventions loaded by ALL agents:
- Header declaring this is for Odoo 17.0 ONLY
- Naming conventions: module name (snake_case), model _name (dot notation), field names (snake_case), method names (_compute_xxx, _check_xxx, action_xxx)
- Import pattern: `from odoo import api, fields, models` (never `from openerp`)
- Python style: 120 char line length (OCA), import ordering (stdlib, odoo, third-party), no `@api.multi`/`@api.one`/`@api.returns`
- File organization: OCA directory structure (models/, views/, security/, tests/, data/, demo/, i18n/, static/description/)
- Version format: 17.0.1.0.0 (5-part)
- License: LGPL-3 for OCA modules
- XML root tag: always `<odoo>` (never `<openerp>`)
- Quick reference of the biggest version pitfalls (attrs removed, states removed, openerp removed)

**knowledge/models.md** (~350-450 lines) -- ORM, fields, decorators, constraints:
- Section: Field Types -- Char, Text, Integer, Float, Boolean, Date, Datetime, Selection, Binary, Html, Monetary with correct usage and common mistakes
- Section: Relational Fields -- Many2one, One2many, Many2many with comodel_name, inverse_name, relation parameters
- Section: Computed Fields -- @api.depends, store=True, compute/inverse pattern, field definition before compute method
- Section: Constraints -- @api.constrains vs SQL constraints, ValidationError, proper self iteration
- Section: CRUD Overrides -- create/write/unlink with super() calls, @api.model on create
- Section: Decorators -- @api.depends, @api.constrains, @api.onchange, @api.model, @api.model_create_multi (NO @api.multi, @api.one, @api.returns)
- Section: Inheritance -- _inherit (extend existing), _name + _inherit (delegation), _inherits
- Section: OCA Conventions -- one model per file, _description required, _order recommended, _rec_name if not 'name'
- Section: Changed in 17.0 -- what-was/what-is table
- Section: pylint-odoo Rules -- W8120, W8150, W8105, R8110 with explanations
- Every rule follows: Rule statement, WRONG code, CORRECT code, Why explanation
- Each code example must be complete enough to copy-paste (imports included where relevant)

Do NOT exceed 500 lines per file.

<task type="auto">
  <name>Task 2: Create views.md, security.md, and manifest.md</name>
  <files>knowledge/views.md, knowledge/security.md, knowledge/manifest.md</files>
  <action>
Create three category files:

**knowledge/views.md** (~350-450 lines) -- XML view rules:
- Section: Form Views -- `<form>`, `<header>`, `<sheet>`, `<group>`, `<notebook>`, `<chatter/>` structure
- Section: List/Tree Views -- Use `<tree>` (NOT `<list>` in 17.0), `column_invisible`, `editable`, `decoration-*`
- Section: Search Views -- `<search>`, `<field>`, `<filter>`, `<group>`, default filters
- Section: View Modifiers -- `invisible="expression"`, `readonly="expression"`, `required="expression"` (NOT attrs dict)
- Section: Actions and Menus -- `ir.actions.act_window`, menu hierarchy, action binding
- Section: External IDs -- naming convention: `{model_underscore}_view_{type}`, `{model_underscore}_action`, `menu_{model_underscore}`
- Section: Statusbar Pattern -- `widget="statusbar"`, `statusbar_visible`, button `invisible` expressions
- Section: Changed in 17.0 -- attrs removed, column_invisible new, states removed, inline expressions
- Section: pylint-odoo Rules -- W8140 with explanation
- Every rule: Rule statement, WRONG XML, CORRECT XML, Why

**knowledge/security.md** (~250-350 lines) -- ACLs, groups, record rules:
- Section: Module Category -- `ir.module.category` record for grouping
- Section: Security Groups -- User and Manager groups, `implied_ids` chain, `category_id` reference
- Section: Access Control Lists -- CSV format, column names, model reference format (`model_{dots_to_underscores}`), permission patterns (user: RWC, manager: RWCU)
- Section: Record Rules -- domain syntax, multi-company rule pattern, user-specific rules, global rules
- Section: Common Mistakes -- missing ACL for a model, wrong model reference in CSV, flat group hierarchy (no implied_ids), groups defined after ACLs in data list
- Section: Data Load Order -- security groups XML loaded before ACL CSV in manifest `data` list
- Section: Changed in 17.0 -- any security-related changes
- Section: pylint-odoo Rules -- W8180 with explanation

**knowledge/manifest.md** (~200-250 lines) -- `__manifest__.py` rules:
- Section: Required Keys -- name, version, category, summary, license, depends, data, installable
- Section: Version Format -- `17.0.1.0.0` pattern, explanation of each part
- Section: License -- `LGPL-3` for OCA, other valid options
- Section: Forbidden/Deprecated Keys -- `description` (use README.rst), `auto_install` (usually wrong)
- Section: Data and Demo Lists -- correct file paths, load order (security before views)
- Section: Dependencies -- explicit is better than implicit, base is always implicit
- Section: Application Flag -- when to set `application: True`
- Section: Changed in 17.0 -- any manifest-specific changes
- Section: pylint-odoo Rules -- E8501, C8101, C8104, C8108 with explanations

Do NOT exceed 500 lines per file.
