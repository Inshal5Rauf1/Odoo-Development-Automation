    Agent role and workflow:
    - **Phase 1: Search** — Accept user's search query (from $ARGUMENTS or existing spec.json). Run `odoo-gen-utils search-modules "<query>"` to get 5 results. If no OCA results, automatically retry with `--github` flag. Present results per Decision A format: name + score + OCA badge + summary + ~coverage%.
    - **Phase 2: Selection** — User picks a result to analyze, or types "build from scratch" to skip fork flow. Each follow-up query re-runs search-modules independently (no session state, per Decision A).
    - **Phase 3: Gap Analysis (SRCH-04, REFN-02)** — For the selected module:
      1. Clone module using git sparse checkout: `git clone --no-checkout --filter=blob:none --sparse -b 17.0 https://github.com/OCA/{repo}.git /tmp/oca_{module} && git -C /tmp/oca_{module} sparse-checkout set {module} && git -C /tmp/oca_{module} checkout 17.0`
      2. Read __manifest__.py, scan models/ for _name and field definitions, scan views/ for view types, check security/ for groups
      3. If user has a spec.json: compare spec fields/models vs module's actual structure
      4. Produce structured gap analysis:
         - **Covered:** models, fields, views, security already in the module
         - **Missing:** spec requirements not in the module
         - **Conflicts:** architectural mismatches (different base model, incompatible inheritance)
      5. Present coverage estimate: "~X% of your spec is covered by this module"
    - **Phase 4: Spec Refinement (REFN-01, REFN-03)** — If user wants to fork:
      1. Present the gap analysis with recommendation: if <40% coverage, suggest building from scratch
      2. Let user adjust the spec based on what exists (REFN-01)
      3. Highlight covered vs gaps clearly (REFN-02)
      4. Save refined spec to `{module_name}_ext/spec.json` AND overwrite the original spec.json path (REFN-03: refined spec is the new source of truth for all downstream generation commands including render-module, validate, etc.)
      5. The refined spec excludes already-covered fields and focuses on the delta
    - **Phase 5: Decision (SRCH-05)** — User picks: "fork and extend" (triggers /odoo-gen:extend) or "build from scratch" (triggers /odoo-gen:new or /odoo-gen:plan)
